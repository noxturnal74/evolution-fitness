# Audit Before — Evolution Fitness Static Gym Pages

## Project Summary
- **Project Type**: Static HTML/CSS/JS Directory & Multi-Gym Landing Pages
- **Total Generated Pages**: 27 gym pages + Homepage + Hub page (29 total HTML documents)
- **Main Generator**: `scripts/generate-20-gym-sites.mjs`
- **Shared CSS**: `gyms/shared/site.css` (generated by the generator script)
- **Shared JS**: `gyms/shared/site.js` (generated by the generator script)
- **Current Live URL**: `https://project-xc6z6.vercel.app`

## Current Architecture
- **Generator Role**: Reads `gyms/gyms.config.json` configuration data, builds 27 independent landing pages (slug/index.html & gyms/slug/index.html), builds directories (`index.html` and `gyms/index.html`), and compiles minified/patched shared CSS & JS assets.
- **Shared CSS Role**: Serves as the global design system. Contains core resets, tokens, component layouts, and color-theme patches for dark-hero / light-body breathable contrast.
- **Shared JS Role**: Handles core interaction (mobile nav, filters, lightbox, accordion, sticky CTA) and GSAP/ScrollTrigger animation binding.
- **Build Checker Role**: Validates SEO tags, landmarks, local refs, banned public texts (e.g. "placeholder", "dummy", "localhost"), GSAP inclusions, and structural section integrity.

## UI/UX Problems
| Area | Problem | Impact | Priority |
|---|---|---|---|
| Color Contrast | Some light sections look washed out, and some dark elements on warm backgrounds lack contrast. | Lower readability and poor accessibility. | High |
| Harlequins / Neon | Tone is slightly neon-heavy with orange and green brand hues on pure dark in some spots. | Feels like a gaming website rather than a premium professional fitness site. | Medium |
| Layout Transitions | Harsh transition between dark cinematic hero `#151515` and plain light body `#f7f6f2`. | Jarring visual flow on scroll. | Medium |
| Card Spacing | Cards are compact with shadows that can feel muddy. | Visual clutter, less breathable than ideal premium directory design. | Medium |
| Mobile Hierarchy | Header and sticky CTA elements can stack awkwardly on small screen widths. | Compromised mobile experience for touch-targeted actions. | High |

## Code Quality Problems
| File | Problem | Risk | Fix Plan |
|---|---|---|---|
| `generate-20-gym-sites.mjs` | Massive minified CSS & JS templates written as hard-coded strings. | Extremely hard to read, maintain, or refactor. | Refactor templates into readable, well-commented blocks separated by component groups. |
| `gyms/shared/site.css` | Raw minified string output with no comments or section headings. | Violates clean coding practices and prevents easy debugging. | Restructure generator output with clean section headers and proper indentation. |
| `gyms/shared/site.js` | JS is minified directly in the generator template. Lack of extensive safety guards. | Console errors if specific selectors are missing, and hard to inspect. | Format output, verify query selectors, and wrap in defensive initialization blocks. |

## SEO Problems
| Page/Template | Problem | Impact |
|---|---|---|
| Schema.org JSON-LD | Structured data is present but doesn't fully represent local business specificity (e.g. lack of exact coordinates, price range indicators). | Missed opportunities for advanced local SEO rich snippets. |
| Alt Text | Images contain generic alt descriptions (`alt="Ilustrasi suasana latihan gym"`). | Less descriptive for search crawlers and screen readers. |

## Accessibility Problems
| Component | Problem | WCAG Concern | Fix |
|---|---|---|---|
| Keyboard Navigation | Mobile menu and lightbox modal elements are not fully keyboard navigable (focus trapping, focus state indicators). | WCAG 2.2 Level AA Keyboard Access | Enhance event listeners, add focus trapping hooks, and verify `:focus-visible` styling. |
| Heading Hierarchy | Some generated sub-sections do not strictly follow logical order (`H1` -> `H2` -> `H3`). | WCAG 1.3.1 Info & Relationships | Standardize headings across generated templates. |

## Performance Problems
| Asset/Component | Problem | Impact | Fix |
|---|---|---|---|
| GSAP Animations | Heavy parallax and ScrollTrigger bindings run continuously. | Potential lag/jank on lower-end mobile devices. | Optimize animations, throttle scroll events, and implement proper guard checks. |
| Asset Preloading | Hero images rely on Unsplash redirects, causing high latency. | Higher LCP (Largest Contentful Paint) times. | Ensure `fetchpriority="high"` and add preconnect/dns-prefetch tags for unsplash domain. |

## Placeholder / Data Risk
| Data | Location | Risk | Action |
|---|---|---|---|
| Banned Text | Public files could accidentally contain placeholders. | Broken build or unprofessional output. | Enhance `build-check.mjs` to block advanced placeholder variations and check structural content validity. |
